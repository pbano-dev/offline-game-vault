# Repository and delivery hygiene

Generated and temporary files are forbidden in source-control commits and in
repo-ready delivery archives.

## Enforced paths

The checker rejects, among others:

```text
__pycache__/
*.pyc
*.pyo
*.egg-info/
.pytest_cache/
.mypy_cache/
.ruff_cache/
build/
dist/
.venv/
venv/
*.tmp
*.temp
*.bak
*.orig
*.rej
*~
.incoming-*
.DS_Store
Thumbs.db
.coverage*
```

## Local checks

Tracked files:

```bash
PYTHONDONTWRITEBYTECODE=1 python -B tools/check_repository_hygiene.py --tracked
```

Complete work tree:

```bash
PYTHONDONTWRITEBYTECODE=1 python -B tools/check_repository_hygiene.py --filesystem
```

The complete-tree scan excludes `.git/` internals.

## CI sequence

1. Check tracked paths before installing dependencies.
2. Do not install the package in editable mode.
3. Set `PYTHONDONTWRITEBYTECODE=1`.
4. Run Python with `-B`.
5. Scan the complete work tree after all tests.

The final scan detects files generated by the validation process itself.

## Delivery rule

Every repo-ready ZIP must be scanned in two forms:

1. the source directory before compression;
2. a freshly extracted copy of the ZIP.

A delivery is invalid if either scan reports a forbidden path.
